Data Quality Assessment and Summarization Tool

This project involves building a Streamlit-based tool that serves two primary purposes:1. PDF Analyzer 2. Data Quality Evaluation Tool. Technologies used are Python, Amazon Web Services (AWS), PyPdf , Nougat , Spacy , Pandas Profiling , Great Expectations

Project Details

Overview

1. PDF Analyzer: The aim is to build a tool that will load a PDF link provided by the user and extract the data from the PDF.

2. Quality Assessment Tool: The aim is to build a tool that takes the file containing the Freddie Mac dataset that needs to be evaluated and tests the data quality. It is checked if the dataset adheres to the schema and follows great expectations.

Part 1: PDF Analyzer

  • Allows users to input a link to a PDF document from the SEC website.
  • Provides the option to choose between the "nougat" and "pypdf" libraries for PDF processing.
  • The two data sources (PDF files) are connected directly to the "streamlit" component.
  • From "streamlit," the data flows to two different custom components: "Nougat" and "PyPdf."
  • Both "Nougat" and "PyPdf" components feed their processed data to a custom component called "Spacy."

Part 2: Data Quality Evaluation Tool

  • Data from the "Freddie Mac Origination File" and "Freddi Mac Monthly Performance File" is directed to the "Pandas Profiling" component for data processing.
  • Enables users to upload CSV/XLS files containing either Origination or Monthly performance data from the Freddie Mac Single Family Dataset.
  • Pandas Profiling is performed to generate data summaries and displays them to the end user.
  • Great Expectations tests are executed to ensure that the data adheres to the schema published by Freddie Mac by performing validations. This validation process ensures that the data meets the specified criteria and quality standards.
